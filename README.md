# test-project

这是一个可在 **微信开发者工具** 中直接编译运行的 **微信小程序版本** 原型，用来模拟：

- 展示一篇微信公众号文章
- 在右下角显示一个类似苹果风格的圆形悬浮按钮
- 点击后把当前文章加入“已收录消息”
- 支持“今日整理 / 本周脉络”切换
- 自动生成整理摘要
- 输出更有结构的总结内容，而不只是简单列主题
- 支持“真实文章摘要化”和“多篇文章串联”两类输出

## 这次增强了什么

现在整理结果不仅有总摘要、核心主题、关键观点，还新增了两种更贴近真实使用场景的输出：

### 1. 真实文章摘要化

每篇文章都会被整理成 3 个稳定结构：

- **问题**：这篇文章在解决什么问题
- **方法**：作者提出了什么思路/路径
- **价值**：这篇文章为什么值得保留

这样你回看时，不必重新读整篇文章，也能快速理解它的意义。

### 2. 多篇文章串联

当一周内出现多篇相近主题的文章时，系统会进一步生成一条串联路径：

- 先做什么
- 中间如何归并主题
- 最后怎样把多篇文章压缩成你的周报/行动方向

这样输出会更像真正的“知识线索”，而不是一堆并列收藏。

## 运行方式

1. 打开微信开发者工具。
2. 导入当前仓库目录。
3. 使用测试号或你自己的小程序 AppID。
4. 编译后即可看到首页。

## 当前能实现的整理效果

点击右下角按钮把文章加入助手后，小程序现在可以：

- 自动把文章纳入当天或本周的整理范围。
- 为单篇文章输出“问题 / 方法 / 价值”摘要卡。
- 提炼当前批次内容的核心主题。
- 总结这批文章里重复出现的关键观点。
- 解释这些文章为什么值得放在一起看。
- 生成多篇文章的串联路径。
- 给出下一步的复盘/行动建议。
- 保留来源线索，方便你回看原始文章。

## 说明

- 当前仓库提供的是 **小程序界面与交互原型**。
- 页面不使用 `document`、`createElement`、`addEventListener` 等网页 DOM API。
- 所有交互通过小程序的 `Page`、`setData`、`bindtap`、WXML/WXSS 实现。
