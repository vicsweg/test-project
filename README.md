# test-project

这是一个 **微信小程序原型**：把公众号文章链接导入到小程序里，再按“当天 / 本周”自动归类，最后生成一段适合发给自己或粘贴到“文件传输助手”的整理文本。

## 这次为什么重做

你明确说“**我需要可以在微信使用**”。

所以这次不再提供单独 HTML 原型，而是改成了 **微信小程序版本**，因为：

- 小程序可以直接在微信里运行。
- 小程序可以读取/写入剪贴板，适合“复制公众号链接 → 导入 → 整理 → 再复制结果”。
- 但 **标准微信能力不能直接给公众号文章详情页注入一个自定义悬浮按钮**。
- 同样，**标准小程序也不能直接控制微信客户端里的“文件传输助手”聊天会话并替你发送消息**。

因此，这个原型采用的是 **微信内可落地的替代方案**：

1. 在微信里复制公众号文章链接。
2. 打开小程序，把链接“收进助手”。
3. 小程序按天/按周整理内容。
4. 一键复制整理结果。
5. 你再粘贴到“文件传输助手”或转发给自己。

## 仓库结构

- `app.js` / `app.json` / `app.wxss`：小程序应用入口配置。
- `pages/index/*`：主页面，负责导入文章、展示收录列表、切换日/周整理、复制整理结果。
- `utils/articles.js`：文章导入、时间格式化、主题统计、整理文案生成。
- `project.config.json`：微信开发者工具项目配置。

## 如何运行

1. 打开 **微信开发者工具**。
2. 选择“导入项目”。
3. 项目目录选择当前仓库根目录。
4. `AppID` 可先使用开发者工具里的测试号（仓库里已写 `touristappid` 用于原型）。
5. 进入后直接预览首页。

## 当前原型能力

- 粘贴公众号文章链接。
- 一键填入演示链接。
- 把文章收进助手收件箱。
- 按“今日整理 / 本周脉络”切换视图。
- 自动提取主题并生成线索列表。
- 一键复制整理结果，方便再发给自己。
- 支持小程序转发卡片入口。

## 如果你要真正做成“更像文件传输助手”的产品

后面建议走这个方向：

1. **小程序前端**：负责导入链接、展示收件箱、触发整理。
2. **后端服务**：抓取文章元信息、摘要、标签、向量化聚类。
3. **消息回传层**：通过企业微信、服务通知、订阅消息或你的私有面板回传整理结果。
4. **个人工作流联动**：把整理结果同步到 Notion、飞书、多维表、Obsidian 等。

## 重要限制说明

> 这个仓库当前做的是“微信里可以运行”的原型，不是对公众号文章详情页的真实注入。
>
> 如果你的目标是“在公众号文章页里就有一个按钮，点一下自动发进文件传输助手”，那通常已经超出标准微信开放能力，需要依赖非常规客户端改造、浏览器插件或其他不稳定/不合规方案。
